# Deep Forest — 遊戲設計文件 (Game Design Document)

> **遊戲名稱**：Deep Forest
> **遊戲類型**：單機 Roguelike 卡牌遊戲
> **畫面風格**：ASCII Art（黑底 + 輻射綠色調）
> **主題基調**：懸疑、微恐、生存
> **引擎**：Godot 4.6 Mono (C#)

---

## 一、遊戲概述

### 1.1 遊戲目標

玩家操控一名受困於詭異森林中的角色，核心目標是：

- **盡可能延長存活天數**
- **蒐集各種物品**
- **透過反覆遊玩，以不同結局解鎖角色的背景故事片段，拼湊出角色進入森林的完整真相**

### 1.2 背景故事

森林是一個神秘而危險的地方。每個可選角色都有各自進入深林的理由。玩家在森林中面對野獸、怪物、未知地形與超自然現象，一邊生存一邊蒐集線索，最終迎向多種可能的結局。

## 二、視覺風格與色彩系統

### 2.1 整體風格

- 以 **ASCII Art** 方式呈現所有畫面
- 視覺風格參考**日本 2ch AA（Shift-JIS Art）**的線條繪畫風格，利用字元的**形狀**勾勒輪廓與表情，而非歐美「亮度對應（Density-based）」的灰階映射
- 所有 ASCII Art 使用**等寬字型（Monospace Font）**繪製，確保字元可作為網格座標精確定位與替換
- 渲染核心為 **Text-Grid Rendering（文字網格渲染）**系統：以 C# 二維字元陣列為畫布，透過座標進行拼貼與動態替換，最終輸出至 Godot `RichTextLabel` 顯示
- 黑底、白線框、輻射綠主色調，營造老式螢幕的詭異氛圍
- **不使用 3D 場景 + Post-Processing Shader 方案**，所有視覺內容均為純文字渲染

### 2.2 色彩定義表

| 名稱 | Hex 參考值 | 用途 |
|------|-----------|------|
| **黑底** | `#000000` | 全畫面背景 |
| **白線** | `#FFFFFF` | 框線、分隔線、面板邊界 |
| **輻射綠** | `#39FF14` | 預設文字、基本 UI、可選行動 |
| **鮮紅** | `#FF2222` | 危險警告、受傷、敵人、條件未滿行動 |
| **亮藍** | `#00BFFF` | 特殊互動、水源、提示 |
| **暗紅** | `#660000` | 穢祟暗示（黑底上幾乎不可讀，需仔細盯著才能看見） |
| **深藍** | `#000066` | 夢境、隱藏線索（黑底上幾乎不可讀） |
| **灰綠** | `#556B2F` | 點數不足行動、已消耗資源、禁用元素 |

### 2.3 色彩運用規則

- **正常狀態**：輻射綠文字 + 白線框 + 黑底
- **受傷/危險**：鮮紅文字閃爍
- **水/冰/特殊互動**：亮藍文字
- **行動可選**：輻射綠（▶ 標記）
- **行動點數不足**：灰綠（▽ 標記，如 `力量 2/5`）
- **行動條件未滿**：鮮紅（✗ 標記，如 `需要：水瓶`）
- **暗紅文字**：穢祟暗示——在 ASCII 場景角落以幾乎不可見的暗紅字元拼出隱藏訊息
- **深藍文字**：夢境場景、隱藏線索，同樣不易讀

### 2.4 字型與字元規範

#### 2.4.1 等寬字型 (Monospace Font)

- 所有文字與 ASCII Art 渲染使用**同一套等寬字型**
- 字型必須滿足以下條件：
  - 半形字元（ASCII `!` ~ `~`）等寬
  - 全形字元（如中文）寬度恰好為半形的 **2 倍**
  - 特殊繪圖符號（Box Drawing `─│┌┐└┘├┤┬┴┼` 等）正確對齊
- 推薦字型：**Sarasa Mono TC（更紗等幅）** 或 **Noto Sans Mono CJK TC**

#### 2.4.2 空白格佔位規則

- ASCII Art 中的「空白區域」根據語境選擇不同的佔位方式：
  - **背景空白**：使用半形空格 ` `（預設黑底不可見）
  - **需要視覺辨識的空白**：使用半形中黑點 `·`（U+00B7）或全形空格 `　`
  - **隱藏字（暗色字）**：使用與繪製字元**等寬**的字元，以暗紅 `#660000` 或深藍 `#000066` 著色，確保佈局不因隱藏而錯位
- **關鍵約束**：隱藏字的字元寬度**必須**與該位置原本應有的字元寬度一致（半形對半形、全形對全形），否則整行字元網格會錯位

#### 2.4.3 CJK 寬度規則

- 半形字元佔 **1 格**
- 全形字元（中文、全形符號）佔 **2 格**
- ASCII Art 模板中如需混用中英文，必須在設計時預留正確的格數
- 例如：`[乾糧]` 佔 6 格（`[` 1 + `乾` 2 + `糧` 2 + `]` 1）

## 四、卡牌系統

### 4.1 核心概念：牌組即背包 (Deck-as-Backpack)

卡組代表角色的旅行背包。卡牌代表資源、行動能力、道具或拖累負擔的障礙物。牌組上限即為角色的負重容量。

### 4.2 卡牌分類

| 類別 | 說明 | 可否主動打出 |
|------|------|:---:|
| **基本行動卡** | 力量 (STR)、靈巧 (DEX)、智慧 (WIS) 三類 | ✅ |
| **道具卡 (消耗品)** | 食物、飲水、藥草、火把等，打出後從牌組中移除 | ✅ |
| **道具卡 (裝備)** | 武器、護甲、工具等，見裝備機制（第五節） | ✅ |
| **被動效果卡** | 依各自條件自動生效，無法主動打出 | ❌ |
| **重要物品** | 劇情關鍵道具，佔手牌空間，影響結局判定 | ❌ |
| **詛咒/異常卡** | 無法打出，佔手牌，產生持續負面效果 | ❌ |
| **傷勢卡** | 受傷時加入牌組，產生持續負面效果 | ❌ |

### 4.3 基本行動卡

- **力量卡 (STR)**：物理行動（劈砍、推動、建造等）
- **靈巧卡 (DEX)**：敏捷行動（閃躲、潛行、搜索等）
- **智慧卡 (WIS)**：觀察與理性（分析、辨識、保持冷靜等）

### 4.4 出牌 Cost 機制

> **打出牌 = 進行行動。每張卡牌都有獨立的 Cost 定義。**

每張卡的 Cost 是一組資源消耗的組合：

| 可消耗資源 | 說明 |
|-----------|------|
| `hunger_cost` | 消耗飢餓值（所有行動卡都會消耗） |
| `thirst_cost` | 消耗口渴值（部分卡牌額外消耗） |
| `hp_cost` | 消耗體力（極端或危險行動） |
| `sanity_cost` | 消耗理智（面對恐怖、禁忌知識） |

### 4.5 卡牌範例

| 卡牌名稱 | 類別 | 效果 | Cost |
|----------|------|------|------|
| `[劈開障礙]` | STR 行動 | +3 力量值 | 飢餓 -2 |
| `[悄悄前進]` | DEX 行動 | +2 靈巧值 | 飢餓 -1 |
| `[全力衝刺]` | DEX 行動 | +4 靈巧值 | 飢餓 -2, 口渴 -2 |
| `[保持冷靜]` | WIS 行動 | +2 智慧值 | 飢餓 -1 |
| `[凝視深淵]` | WIS 行動 | +5 智慧值 | 理智 -3 |
| `[乾糧]` | 消耗品 | 飢餓 +8 | 口渴 -3 |
| `[水壺]` | 消耗品 | 口渴 +10 | — |
| `[草藥]` | 消耗品 | 體力 +5 | 飢餓 -1 |

## 五、裝備機制

### 5.1 核心概念

裝備卡在行動階段打出後，**離開牌組**並帶來持續效果。同時會在牌組中加入一張通常**等重**的 `[卸下 XXX]` 卡，以及可能附帶的使用卡。

### 5.2 裝備流程

1. 手牌中有裝備卡（例如 `[短劍]`，重量 2）
2. 打出裝備 → `[短劍]` 離開牌組，進入裝備欄
3. 裝備效果生效（例如 STR 行動 +1 傷害）
4. 牌組中加入 `[卸下短劍]`（重量 2）+ `[劍擊]`（重量 0）
5. 若打出 `[卸下短劍]` → 短劍回到牌組，移除卸下卡和使用卡

### 5.3 設計要點

- **重量守恆**：裝備離開牌組，但卸下卡佔據等重空間，不會因裝備而憑空騰出容量
- **使用卡**：部分裝備附帶的使用卡（如 `[劍擊]`）重量為 0，是裝備帶來的額外行動能力
- **卸裝策略**：緊急時可卸下裝備來騰出空間放入新物品，但失去裝備效果

### 5.4 裝備範例

| 裝備卡 | 重量 | 裝備效果 | 加入牌組的卡 |
|-------|:---:|---------|------------|
| `[短劍]` | 2 | STR 行動 +1 傷害 | `[卸下短劍]`(重量 2) + `[劍擊]`(重量 0, STR +4, 飢餓 -3) |
| `[皮甲]` | 3 | 受傷時體力損失 -2 | `[卸下皮甲]`(重量 3) |
| `[水瓶]` | 1 | 解鎖「裝水」行動 | `[卸下水瓶]`(重量 1) |
| `[火把]` | 1 | 夜間照明，搜索 WIS +1 | `[卸下火把]`(重量 1) + `[照明]`(重量 0, WIS +1, 消耗耐久) |
| `[背包]` | 2 | 牌組上限 +5 | `[卸下背包]`(重量 2) |

## 六、角色數值系統

### 6.1 明示量表（顯示在左側狀態面板）

| 量表 | 說明 | 歸零後果 |
|------|------|---------|
| **體力 (HP)** | 生命值。受傷、完成任務時扣減；換日時恢復 | **遊戲結束** |
| **理智 (Sanity)** | 精神狀態。受傷、特定卡牌 Cost 時扣減；換日時恢復 | 觸發幻覺、視覺異常、強制事件 |
| **飢餓 (Hunger)** | 食物存量。打出行動卡（依 Cost）和換日時消耗；打出食物卡恢復 | 每回合開始扣體力 + 理智 |
| **口渴 (Thirst)** | 水分存量。打出特定卡（依 Cost）和換日時消耗；打出飲水卡恢復 | 每回合開始扣體力 + 理智 |

**關鍵規則**：食物和飲水**不會被動消耗**。玩家需主動從手牌中打出食物/飲水卡來恢復飢餓/口渴值。飢餓或口渴歸零後才開始每回合被動扣除體力和理智。

### 6.2 資源消耗時機一覽

| 觸發條件 | 消耗/恢復的資源 |
|----------|---------------|
| 打出行動卡 | 依該卡的 Cost（飢餓、口渴、體力、理智的組合） |
| 打出消耗品 | 恢復對應資源，部分道具有反向消耗（如乾糧：飢餓 +8, 口渴 -3） |
| 完成任務 | 體力 -N（依任務難度） |
| 受到傷害 | 體力 -N，理智 -M |
| 換日 | 飢餓 -N，口渴 -M（依天數/深度遞增）；體力 +X，理智 +Y（依環境品質） |
| 飢餓 = 0 | 每回合開始扣體力和理智 |
| 口渴 = 0 | 每回合開始扣體力和理智 |

### 6.3 隱藏數值（不顯示數字，透過敘事暗示）

三個隱藏數值分別驅向三種不同的結局路線：

#### 暴戾 (Brutality) → 偏激暴力路線

| 等級 | 閾值 | 效果 |
|------|------|------|
| 低 | 0~20 | 正常選項 |
| 中 | 21~50 | 事件中額外出現偏激選項（如「直接殺掉」），結果伴隨高傷害但自損 |
| 高 | 51~80 | 偏激選項增多，正常選項描述帶攻擊性語氣 |
| 極高 | 81+ | 部分溫和選項消失，走向暴力結局 |

**暗示方式**：敘事文字語氣變化、NPC 對話態度轉變

#### 穢祟 (Corruption) → 克蘇魯式結局

| 等級 | 閾值 | 效果 |
|------|------|------|
| 低 | 0~20 | 正常 |
| 中 | 21~50 | 掉理智事件頻率增加，換日夢境出現詭異內容 |
| 高 | 51~80 | ASCII 場景中以暗紅色浮現不可名狀的符號，牌組生出無法移除的 `[低語]` 詛咒卡 |
| 極高 | 81+ | 場景描述被不可理解的敘事取代，高機率進入克蘇魯式結局 |

**暗示方式**：ASCII 場景中出現不該存在的字元/圖形、重要物品欄出現 `???` 物品、詭異夢境

#### 邪惡 (Evil) → 刑偵結局

| 等級 | 閾值 | 效果 |
|------|------|------|
| 低 | 0~20 | 正常 |
| 中 | 21~50 | 事件中出現犯罪選項（偷竊、欺騙、陷害），帶來短期利益 |
| 高 | 51~80 | NPC 不信任，某些合作選項消失 |
| 極高 | 81+ | 場景中出現調查痕跡，高機率進入刑偵結局 |

**暗示方式**：角色描述措辭微妙改變、善意選項消失

> **混合結局**：三個隱藏值可同時累積。若多個值同時極高（合計 ≥ 120），可觸發混合結局。

### 6.4 角色參數

每個角色擁有以下獨立數值：

| 參數 | 說明 | Default 白板（成年男性） |
|------|------|:---:|
| 抽牌數 (Draw) | 每回合抽幾張牌 | 5 |
| 手牌上限 (Hand Limit) | 回合結束時最多保留幾張 | 7 |
| 牌組上限 (Deck Capacity) | 背包負重 | 30 |
| 體力上限 (Max HP) | — | 100 |
| 理智上限 (Max Sanity) | — | 100 |
| 飢餓上限 (Max Hunger) | — | 50 |
| 口渴上限 (Max Thirst) | — | 50 |
| 起始牌組 | 初始卡牌組合 | 基礎 STR/DEX/WIS 各 4 + 乾糧 ×2 + 水壺 ×2 |

### 6.5 角色衍生管線

1. **Default 白板角色（成年男性）** — 先完成並平衡
2. **調整性別/年齡**：成年女性、老年男/女、幼年男/女
3. **疊加職業**：獵人、學者、醫者等（改變起始牌組 + 賦予獨特被動能力卡）
4. **加上獨特能力 → 進行再平衡**

| 體型/年齡 | 相對白板的調整方向 |
|----------|------------------|
| 成年女性 | 體力↓、靈巧↑、口渴消耗略低 |
| 老年男性 | 體力↓↓、智慧↑↑、抽牌數↑、牌組上限↓ |
| 老年女性 | 體力↓↓、智慧↑、理智↑、牌組上限↓ |
| 幼年男性 | 全數值↓、飢渴消耗低、手牌上限↓ |
| 幼年女性 | 全數值↓↓、靈巧↑、飢渴消耗最低 |

## 七、系統狀態

### 7.1 環境因子

顯示在上方系統橫幅中，影響行動消耗與任務標準：

| 因子 | 可能值 | 影響 |
|------|-------|------|
| **天氣** | 晴朗 / 陰天 / 濃霧 / 暴雨 / 雷暴 | 任務門檻修正（如濃霧 → DEX 需求 +2） |
| **溫度** | 寒冷 / 涼爽 / 溫暖 / 炎熱 / 酷熱 | 口渴消耗修正（如炎熱 → 行動卡口渴 Cost +1） |
| **濕度** | 乾燥 / 適中 / 潮濕 / 極濕 | 休息品質修正（如潮濕 → 體力恢復 -30%） |

### 7.2 環境修正範例

| 環境組合 | 修正效果 |
|---------|---------|
| 濃霧 + 潮濕 | DEX 門檻 +2、休息體力恢復 -30% |
| 炎熱 + 乾燥 | 行動口渴 Cost +1、水源場景出現率↓ |
| 暴雨 + 寒冷 | 換日體力恢復 -50%、無篝火時理智 -3 |
| 晴朗 + 涼爽 | 無修正（基準環境） |

## 八、牌組狀態效果（詛咒/傷勢/心靈創傷）

### 8.1 核心概念

牌組中的詛咒、傷勢、心靈創傷等卡片產生**持續性負面效果**，直到該卡從牌組中消失（被治療、消除、或隨時間自然消失）。

### 8.2 視覺干擾與操作限制

| 卡片 | 效果 | 視覺/操作呈現 |
|------|------|-------------|
| `[幻聽]` | 場景描述中隨機插入不存在的聲音文字 | 隨機文字以暗紅色閃現 |
| `[文字扭曲]` | 畫面中隨機文字 render 時變成亂碼 | 輻射綠字元被亂碼符號取代 |
| `[視野模糊]` | 場景 ASCII art 部分區域模糊化 | 字元被替換為 `░▒▓` |
| `[失明（左眼）]` | 畫面左半部分符號變成黑色 | 左側 ASCII 場景字元漸變為黑色 |
| `[失明（右眼）]` | 畫面右半部分符號變成黑色 | 右側 ASCII 場景字元漸變為黑色 |
| `[手指骨折]` | 無法選擇最左/最右邊的手牌 | 最左/右手牌灰綠色且無法點選 |
| `[手臂脫臼]` | 力量卡效果減半 | STR 行動卡數值以灰綠色顯示 |
| `[恐懼症]` | 遭遇敵人時第一回合自動消耗理智 | 戰鬥開始時畫面短暫全紅閃爍 |
| `[夢魘]` | 換日休息時理智不恢復 | 夜間場景以深藍色呈現詭異文字 |

### 8.3 傷勢與治療

| 傷勢來源 | 加入的卡片 | 消除方式 |
|---------|-----------|---------|
| 戰鬥受傷 | `[擦傷]`、`[割傷]`、`[骨折]` | 消耗品（草藥、繃帶等） |
| 理智過低 | `[幻聽]`、`[恐懼症]`、`[夢魘]` | 理智恢復至一定值後自動消失 |
| 穢祟積累 | `[文字扭曲]`、`[低語]` | 通常**無法移除** |
| 環境傷害 | `[凍傷]`、`[脫水]` | 對應道具或到達安全環境 |

## 九、場景互動系統

### 9.1 核心概念

每個場景是一張 ASCII Art 風格的第一人稱視角圖。場景中包含**數個可執行的事件/行動**，玩家**不需要做完全部**，可以選擇性地完成。

### 9.2 場景互動流程

1. **進入場景** → 顯示 ASCII Art 第一人稱場景圖
2. **行動面板**顯示在場景正中央偏下（不貼齊手牌區），列出可執行的行動與門檻
3. **玩家出牌**：從手牌中打出行動卡，累積 STR/DEX/WIS 數值，每張卡扣除各自的 Cost
4. **行動達標**時，該行動選項由灰綠變為輻射綠（▶ 可選）
5. **玩家點選行動**執行 → 結算效果（消耗體力、獲得獎勵等）
6. 可繼續出牌做其他行動，或直接離開場景

### 9.3 行動狀態指示

| 狀態 | 符號 | 顏色 | 說明 |
|------|------|------|------|
| 可選 | ▶ | 輻射綠 | 累積點數已達門檻 |
| 點數不足 | ▽ | 灰綠 | 門檻類型正確但數值不夠 |
| 條件未滿 | ✗ | 鮮紅 | 缺少必要物品或裝備 |

### 9.4 場景範例

#### 河畔

| 行動 | 需求門檻 | 前置條件 | 效果 |
|------|---------|---------|------|
| 紮營 | 力量 ≥ 4 | — | 觸發換日 |
| 捕魚 | 靈巧 ≥ 3 | — | 獲得 `[生魚]` 卡 |
| 裝水 | — | 已裝備 `[水瓶]` | 獲得 `[清水]` 卡 |
| 前進 | 靈巧 ≥ 2 | — | 離開場景，進入下一節點 |

#### 廢棄獵屋

| 行動 | 需求門檻 | 前置條件 | 效果 |
|------|---------|---------|------|
| 搜索 | 智慧 ≥ 3 | — | 發現 `[生鏽的鑰匙]` 或 `[獵人日記]` |
| 紮營 | 力量 ≥ 2 | — | 觸發換日（良好環境） |
| 撬開地窖 | 力量 ≥ 6 | — | 進入地窖，觸發隱藏事件（可能增加穢祟） |
| 前進 | 靈巧 ≥ 2 | — | 離開場景 |

### 9.5 出牌結算

#### 任務模式
打出行動卡後，依照 STR / DEX / WIS 各類別累積總數值，與行動的通過門檻比較。達標後可選擇執行。

#### 戰鬥模式
依照公式結算每回合傷害（例如：`傷害 = STR 總值 × 武器倍率 - 敵方防禦`），敵方回合同理反擊。

## 十一、回合機制

### 11.1 場景回合

```
1. 回合前延遲效果結算（詛咒卡、傷勢卡、飢渴歸零懲罰）
2. 回合開始
3. 抽牌階段（抽 Draw 張牌）
4. 角色出牌（可決定排序，每張卡扣除各自 Cost，累積 STR/DEX/WIS）
5. 出牌效果結算
6. 牌組整理階段（丟棄超出手牌上限的牌）
7. 任務標準結算（達標 → 消耗體力，進入下一節點）
```

### 11.2 戰鬥回合

```
1. 回合前延遲效果結算
2. 回合開始
3. 抽牌階段
4. 角色出牌（可決定排序，每張卡扣除各自 Cost）
5. 出牌效果結算（傷害計算）
6. 敵方角色狀態更新
7. 牌組整理階段
8. 敵方角色回合（受傷 → 消耗體力 + 理智）
```

### 11.3 換日結算（抽牌堆耗盡 或 主動紮營）

```
1. 消耗飢餓和口渴（依天數 + 深度遞增）
2. 恢復體力和理智（依休息環境品質 × 天氣修正）
3. 天數 +1
4. 棄牌堆 + 手牌 → 洗回抽牌堆
5. 檢查隱藏屬性 → 觸發夢境事件（若符合條件）
```

### 11.4 換日消耗（飢餓 + 口渴，依天數/深度遞增）

| 天數 | 深度 | 飢餓消耗 | 口渴消耗 |
|------|------|---------|---------|
| 第 1~3 天 | 淺層 | -3 | -3 |
| 第 4~7 天 | 中層 | -5 | -5 |
| 第 8~12 天 | 深層 | -7 | -8 |
| 第 13+ 天 | 最深層 | -10 | -12 |

### 11.5 換日恢復（體力 + 理智，依環境品質）

| 環境品質 | 體力恢復 | 理智恢復 | 條件 |
|---------|---------|---------|------|
| 良好 | +15 | +10 | 安全營地、有篝火、獵屋 |
| 普通 | +8 | +5 | 一般野外紮營 |
| 惡劣 | +3 | +0 | 危險區域、天氣惡劣 |
| 詭異 | +5 | **-5** | 穢祟高的區域 |

### 11.6 一天的流逝

- **抽完牌堆** = 一天過去，自動進入換日結算
- **主動紮營** = 非戰鬥狀態下，玩家可選擇主動洗牌（紮營），立刻進入換日結算
- 天數推進會增加換日的飢渴消耗

## 十二、ASCII Art 渲染管線 (Rendering Pipeline)

> 本節定義遊戲中所有 ASCII Art 視覺元素的技術規格與製作流程。

### 12.1 Text-Grid 渲染系統（核心架構）

#### 12.1.1 字元緩衝區 (Character Buffer)

- 遊戲的每個視覺區域（頭像、場景、地圖等）各自維護一個**二維字元陣列**（`CharCell[,]`）
- 每個 `CharCell` 包含：
  - `char Character`：顯示的字元
  - `Color ForegroundColor`：前景色（預設：輻射綠）
  - `Color BackgroundColor`：背景色（預設：黑色 `#000000`）
  - `string Tag`：區域標籤（用於特效定位，見 12.1.3）
  - `bool IsTransparent`：是否為透明格（拼貼時不覆蓋下層）
- 渲染流程：`C# 二維陣列 → BBCode 字串 → RichTextLabel 顯示`

#### 12.1.2 渲染順序（圖層堆疊）

```
底層 → 頂層：
1. 背景填充（空格或 · 佔位）
2. 場景模板（預製文字區塊）
3. 動態貼圖（子圖覆蓋至指定座標）
4. 特效疊加（隱藏字、亂碼、色彩變換）
5. BBCode 輸出
```

- 拼貼規則：上層的非透明格覆蓋下層；標記為 `IsTransparent = true` 的空格不覆蓋

#### 12.1.3 區域標籤系統 (Zone Tag System)

每個 `CharCell` 可被標記一個 `Tag`，用於特效系統精確定位要進行字元替換或變色的區域：

**頭像區域標籤：**

| Tag | 說明 |
|-----|------|
| `avatar.head` | 頭部輪廓 |
| `avatar.face` | 五官（眼、鼻、口） |
| `avatar.body` | 衣著 / 身體 |
| `avatar.bg` | 頭像背景 |

**場景區域標籤：**

| Tag | 說明 |
|-----|------|
| `scene.bg` | 場景背景（天空、遠景） |
| `scene.left` | 左側物件 |
| `scene.center` | 中央主體物件 |
| `scene.right` | 右側物件 |
| `scene.ground` | 下方地面 |
| `scene.overlay` | 覆蓋層（霧氣、雨滴等動態效果） |

**地圖區域標籤：**

| Tag | 說明 |
|-----|------|
| `map.explored` | 已探索節點與路徑 |
| `map.current` | 當前位置 |
| `map.fog` | 未探索的迷霧區域 |

> **用途範例**：低理智效果觸發時，特效系統可查找所有 `Tag == "scene.bg"` 的格子，隨機將部分字元替換為亂碼；穢祟效果可在 `Tag == "scene.overlay"` 的格子中以暗紅色插入隱藏符號。

---

### 12.2 人物頭像 (Character Avatar)

#### 12.2.1 製作流程

```
手繪草稿 → AI 生成參考圖 → 手工/輔助工具轉為等寬 ASCII Art → 存為 .txt 模板
```

#### 12.2.2 頭像規格

| 項目 | 規格 |
|------|------|
| 網格尺寸 | 約 20×16 半形格（可依面板調整） |
| 字型 | 等寬字型，嚴格半形 1 格 / 全形 2 格 |
| 風格 | 2ch AA 線條風格，以符號的**形狀**勾勒輪廓 |
| 常用字元 | `／` `＼` `￣` `＿` `（` `）` `│` `─` `°` `ω` `Д` `∀` 等 |
| 空白佔位 | 背景使用半形空格；需辨識的佔位使用 `·` |
| 標籤分配 | 頭部 / 五官 / 衣著 / 背景 分別標記 Tag |

#### 12.2.3 動態效果

- **受傷**：`avatar.face` 區域的部分字元以鮮紅色疊加血跡符號
- **失明（左眼）**：`avatar.face` 左半側字元被替換為 `■` 或黑色
- **穢祟暗示**：`avatar.bg` 區域以暗紅色隱藏字浮現詭異符號
- **表情變化**：替換 `avatar.face` 中的眼口字元（例如 `°ω°` → `°Д°`）

---

### 12.3 地圖系統 (Map Rendering)

#### 12.3.1 程序化生成 (Procedural Generation)

- 地圖以**節點圖 (Node Graph)** 結構生成，類似《殺戮尖塔》的分支路徑
- 底層資料結構為圖/樹狀結構（C# `Dictionary` 或自定義 `MapNode`）
- 每個節點具有類型標記：

| 符號 | 節點類型 |
|------|---------|
| `◎` | 起始營地 |
| `?` | 未知事件 |
| `!` | 已知危險 |
| `▲` | 紮營點 |
| `★` | 重要地標 |
| `█` | 已探索 |
| `░` | 未探索（迷霧） |

#### 12.3.2 地圖渲染

- 使用 ASCII 字元繪製節點與連線（`─` `│` `┌` `┐` `└` `┘` `├` `┤` `/` `\`）
- 已探索區域以輻射綠顯示
- 當前位置以亮藍色高亮
- 未探索區域以灰綠色亂碼/迷霧覆蓋（`Tag == "map.fog"`）
- 特殊事件節點可閃爍或使用特殊顏色

#### 12.3.3 迷霧效果

- 未探索節點以 `░▒▓` 或隨機亂碼字元覆蓋（灰綠色）
- 玩家前進時，周圍一格範圍的迷霧揭開（字元漸變動畫）
- 高穢祟時，已探索區域也可能重新被迷霧覆蓋

---

### 12.4 場景渲染 (Scene Rendering)

#### 12.4.1 核心策略：預製模板 + 適度動態組合

- **優先預製模板**：每個場景類型預製完整的 ASCII Art 文字區塊（`.txt` 檔案），確保視覺品質
- **輔以動態貼圖**：在模板基礎上，依遊戲狀態在指定座標疊加/替換子圖元素
- **繪製指引**：即使無法預製所有模板，也應建立各場景類型的「繪製指引」（構圖規範、標誌性元素清單）

#### 12.4.2 場景模板分類

| 場景類別 | 說明 | 標誌性元素 |
|---------|------|-----------|
| `forest_path` | 森林小徑 | 兩側樹木、落葉、遠方消失點 |
| `riverside` | 河畔 | 水面波紋 `~≈`、岸邊石頭、對岸樹影 |
| `abandoned_cabin` | 廢棄獵屋 | 屋頂輪廓、門窗、雜草 |
| `cave_entrance` | 洞穴入口 | 岩壁紋理、黑暗深處、苔蘚 |
| `campsite` | 營地 | 帳篷/篝火、背包、周圍樹木 |
| `crossroads` | 分岔路口 | 多條路徑、路標 |
| `dark_clearing` | 陰暗空地 | 開闊天空、枯樹、不祥氛圍 |
| `ruin` | 遺跡 | 殘垣斷壁、石柱、符文 |

#### 12.4.3 場景模板結構

每個場景模板由以下檔案組成：

```
assets/ascii_art/scenes/
├── riverside/
│   ├── base.txt           # 基礎場景模板（完整 ASCII Art）
│   ├── base.tags          # 對應的 Tag 標記檔（與 base.txt 同尺寸）
│   ├── overlay_rain.txt   # 雨天覆蓋層
│   ├── overlay_fog.txt    # 霧天覆蓋層
│   └── sub/               # 可替換子圖
│       ├── campfire.txt   # 營火子圖（貼到 scene.center）
│       ├── fish_trap.txt  # 魚簍子圖（貼到 scene.right）
│       └── tent.txt       # 帳篷子圖（貼到 scene.left）
```

#### 12.4.4 Tag 標記檔格式

- `.tags` 檔案與 `.txt` 檔案逐字元一一對應
- 每個字元位置標記其所屬區域，使用縮寫：

```
B = scene.bg     | L = scene.left    | C = scene.center
R = scene.right  | G = scene.ground  | O = scene.overlay
```

#### 12.4.5 場景拼貼流程

```
1. 載入 base.txt → 填入字元緩衝區
2. 載入 base.tags → 為每個 CharCell 設定 Tag
3. 依遊戲狀態選擇子圖 → 貼到指定 Tag 區域的座標上
4. 依天氣選擇覆蓋層 → 疊加到 scene.overlay 區域
5. 依角色狀態應用特效 → 修改特定 Tag 區域的字元/顏色
6. 輸出 BBCode → 更新 RichTextLabel
```

#### 12.4.6 場景繪製指引

當缺少預製模板時，AI 或美術人員應遵循以下構圖規範：

| 區域 | 位置 | 佔比 | 繪製原則 |
|------|------|:---:|---------|
| `scene.bg` | 上方 1/3 | ~30% | 天空/遠景，使用稀疏字元（`.` `·` `'`） |
| `scene.left` | 左側 | ~20% | 近景物件（樹木、岩石），使用密集字元 |
| `scene.center` | 正中央 | ~30% | 場景主體/焦點物件 |
| `scene.right` | 右側 | ~20% | 近景物件，與左側呼應 |
| `scene.ground` | 下方 1/4 | ~20% | 地面紋理（`_` `~` `.` `▁`） |

### 12.5 恐怖氛圍效果

#### 12.5.1 低理智視覺干擾

| 理智等級 | 效果 | 技術實作 |
|---------|------|---------|
| Sanity < 30% | 場景字元偶爾隨機替換 | 定時器隨機修改 `scene.bg` 中的字元 |
| Sanity < 15% | 色調從輻射綠轉為暗紅，出現幻影字元 | 修改前景色為暗紅，`scene.overlay` 插入閃爍字元 |
| Sanity = 0 | 強制觸發幻覺事件，可能進入無法分辨真假的敘事選擇 | 全場景字元大面積替換 + 事件觸發 |

#### 12.5.2 高穢祟畫面異變

- `scene.overlay` 中以暗紅色（`#660000`）浮現不該存在的圖形（閃爍的眼睛 `◉`、蠕動的符號）
- `map.explored` 中的節點名稱被扭曲字元覆蓋
- `scene.bg` 角落以暗紅色隱藏字拼出若隱若現的訊息

#### 12.5.3 一般氛圍營造

- 正中央 ASCII 場景使用深綠色調
- 遭遇敵人時 `scene.bg` 色調轉為冷灰或血紅
- 霧氣動畫：`scene.overlay` 中的 `░▒` 字元定時平移
- 文字描述搭配氛圍提示（如 `遠處傳來樹枝折斷的聲音...`）

## 十三、結局系統

### 13.1 結局類型

#### 結局 1：逃出森林（物品組合結局）

- **觸發條件**：持有特定物品組合到達遊戲結束
- 不同物品組合對應不同逃出方式與真相揭露
- 角色特有物品組合解鎖該角色核心背景故事

#### 結局 2：人間蒸發（消耗/極限結局）

- **觸發條件**（任一）：
  - 前進到極深深度
  - 存活到極長天數
  - 體力歸零
  - 理智歸零
- 角色消失在森林中，留下物品成為其他角色可能發現的道具

#### 結局 3：隱藏數值結局

| 隱藏值 | 結局 | 描述 |
|--------|------|------|
| 暴戾 ≥ 80 | 暴戾結局 | 角色徹底失控，成為森林中的掠食者 |
| 穢祟 ≥ 80 | 邪祟結局 | 不可名狀的力量吞噬了角色的意識 |
| 邪惡 ≥ 80 | 刑偵結局 | 逃出森林後被逮捕審判 |
| 暴戾 + 穢祟 ≥ 120 | 混合結局 | 被不可名狀力量驅使的屠殺者 |
| 暴戾 + 邪惡 ≥ 120 | 混合結局 | 有意識地利用暴力達成邪惡目的 |
| 穢祟 + 邪惡 ≥ 120 | 混合結局 | 與深淵達成交易，出賣靈魂 |

### 13.2 Roguelike 循環

1. 選擇角色 → 進入森林
2. 探索場景、蒐集物品、延長存活
3. 遊戲結束 → 結局判定（依持有物品、存活天數、隱藏數值）
4. 解鎖該角色的一段背景故事
5. 重複遊玩，嘗試不同路線與結局
6. 所有故事片段解鎖 → 該角色完全通關，拼湊出完整真相
